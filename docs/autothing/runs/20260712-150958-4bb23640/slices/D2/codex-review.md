**Findings**

1. **High: action inputs are not validated against the manifest params.**  
   `extractActions` accepts any object as `input` and attaches the server manifest action without filtering/coercing it against `action.params` ([api/src/apps/app-assistant.ts:224](api/src/apps/app-assistant.ts:224), [api/src/apps/app-assistant.ts:231](api/src/apps/app-assistant.ts:231)). The panel then turns that object into runtime `params` ([api/assets/bases/app/scaffold/frontend/src/lib/assistant/AssistantPanel.jsx:75](api/assets/bases/app/scaffold/frontend/src/lib/assistant/AssistantPanel.jsx:75), [api/assets/bases/app/scaffold/frontend/src/lib/assistant/AssistantPanel.jsx:78](api/assets/bases/app/scaffold/frontend/src/lib/assistant/AssistantPanel.jsx:78)). For `custom` actions, the runtime passes the full params object to app code ([api/assets/action-runtime-client.js:393](api/assets/action-runtime-client.js:393), [api/assets/action-runtime-client.js:396](api/assets/action-runtime-client.js:396)). The manifest declares a typed param contract ([shared/src/action-manifest.ts:18](shared/src/action-manifest.ts:18), [shared/src/action-manifest.ts:59](shared/src/action-manifest.ts:59)), and `assistant-tools` even builds an `additionalProperties: false` schema ([api/src/apps/assistant-tools.ts:48](api/src/apps/assistant-tools.ts:48), [api/src/apps/assistant-tools.ts:58](api/src/apps/assistant-tools.ts:58)), but that schema is not enforced on the fenced JSON path. This leaves undeclared/custom params forgeable by the model path.
