## Part 1 - Credentials, providers, Supabase retirement

1. **FIXED-8 amendment (auth modes).** "Managed OAuth credentials only - no raw API keys" becomes: credentials are centrally managed (encrypted at rest, never per-user ad hoc, never `~/.claude` fallback), and the chokepoint supports **two auth modes as configuration**: Claude subscription OAuth (current) and Anthropic API key (production/customer path). Mode is per-environment config on the egress module (FIXED-13 already owns provider routing).
2. **Credential machinery simplified.** The multi-subscription rollover model is deleted from the spec: no credential pools, no rotation, no health-scoring, no selection logic. What remains: one credential per environment; proactive refresh before expiry; refresh-and-retry-once on 401; alert on persistent failure. The watchdog reduces to that alert. Chapters 04/05/06 references to OAuth rotation semantics are rewritten accordingly.
3. **P-08 re-resolved: Supabase retired entirely.** The three live tables move to Firestore collections (credentials encrypted with the existing crypto module; `companies` is superseded by the org model of Part 4; installation/license facts fold into the activation model of Part 3). Chapter 04 §4.5 rewritten; chapter 09 §9.7 boot gates re-pointed at Firestore; chapter 10 gains the credentials/installation migration rows and drops the Supabase coexistence mechanics.
4. **License server.** Exploration task against `ekoa-dev`: locate any separate license-server app; inventory its surface and consumers; fold needed routes into `api/` and its data into Firestore; retire the rest. With Part 3, "license" reduces to activation + billing - no license service exists in ekoa-code.

## Part 2 - Persistence and privacy posture

5. **Three-posture policy (normative, chapter 09 + 17).**
   - **Egress: tokenize.** Everything model-bound crosses the chapter 17 pipeline - including memories re-injected by the resolver (they are model-bound text and tokenize with the current session's vault automatically; state this explicitly).
   - **At-rest knowledge (messages, memories, knowledge base): cleartext, protected by scoping** (org + user visibility per Part 4), encrypted at rest, enumerated in the custody map, covered by the erasure workflow.
   - **At-rest operational surfaces (logs, run records): omit or irreversibly redact.** Log discipline: no message bodies, no delegation file paths, no tokens; identifiers, codes, counts only, plus the existing redaction middleware. **P-27 resolved:** executor-face content-bearing run-record fields pass through the detector at persist time with **irreversible redaction** of detected spans - never session tokens (the vault is ephemeral; tokens at rest are permanent noise).
6. **P-12 re-resolved: automatic memory extraction returns, correctly attributed.** Auto-extract ships ON: asynchronous post-run, never adding turn latency; Haiku-class, batched per run; tagged **`user_work`** (it is personalization the user receives and is billable - the zero-platform-calls posture of chapter 06 stands untouched; re-fate call 23 accordingly). Applies to hosted agent runs only - delegated local work is mined solely from derived output already in the hosted record (I2 preserved). Privacy-scrub patterns kept; consolidation stays deterministic code. Per-user toggle, default ON. Every automatic write is visible: a Registo entry and a UI affordance. **Extraction always writes `private`** - sharedness is never inferred (Part 4).

## Part 3 - Activation model (replaces licensing)

7. Two independent facts govern access: **`active`** (boolean, admin-controlled) and **allowance/credits** (the existing billing gate). No tiers; the tier feature map is dropped as a concept. Enforcement: the three admission planes (user JWT middleware, served-app plane gate, bridge pairing) consult a cached activation state - in-memory map with **write-through invalidation** (toggling a user updates the map in the same write; TTL is a safety net only). Deactivation does two things: sets `active=false` AND pushes the user's tokens into the P-03 revocation set. Two CONV-2 error codes with PT-PT copy: `ACCOUNT_DISABLED` ("A sua conta está bloqueada. Contacte o suporte.") and `BILLING_LOCKED` ("A sua conta tem um problema de faturação. Contacte o suporte."). Payment-provider automation is a future webhook flipping the same boolean; note it, build nothing.

## Part 4 - The org model (tenant, finally defined)

8. **Schema.** `orgs` collection; every user carries a required `orgId` at creation. Creating a standalone user auto-creates an org with that user as its admin. Schema/API name: **`org`**; PT-PT display label **"Escritório"** (labels are vertical copy, schema names permanent). `tenant` is defined as `org` in one sweep across the spec - including chapter 17 deny-lists and chapter 18 delegation bindings ({tenant, user, session, pairing, ...} becomes {org, user, session, pairing, ...}).
9. **Org-scoped (the complete list):** branding + brand research (per-org record; admin-only carried; research overwrites only its org); org settings; **the knowledge base** (vault + lexical index org-partitioned - the firm's documents never pool across orgs); **integration credentials** (the M365 workspace token of Q-10 is per-org); anonymisation deny-lists; org-shared memories; org-shared artifacts; automations, triggers, and their runs (visible to owner + org admin); the Registo read surface (Part 5). `design-tokens.css` keeps its URL and byte-contract: the org resolves **server-side from the app's slug** - the 37 legal e2e specs do not move.
10. **User-scoped (unchanged):** sessions, messages, private memories and artifacts, sandboxes, local-bridge pairings, per-user metering and allowance (org-level billing rollup is a later Registo view, not a billing rework).
11. **Permissions.** Three levels, two roles: **super-admin** (platform: global users page, creates orgs/users, activate/deactivate anywhere; seeded as the founder's account at migration), **org admin** (branding/research, org settings, org users page scoped to own org - role toggle and deactivate only, automation creation, integrations config, Registo view), **builder** (chat, builds, own artifacts/memories, use org-shared resources, delegate own local files). Automation creation admin-only is a flippable default.
12. **Sharing enforcement (ownership × visibility).** `visibility: private | org` on memories and artifacts. Private → owner only; invisible to org admins too (existence appears in Registo metadata, never content). Org-shared artifacts: visible and **editable** by org members - safe because git version snapshots + restore + Registo cover every mutation. Org-shared memories: resolver injects own + org-shared. Promotion to org is a manual UI toggle; demotion allowed by owner. Cross-org access impossible by construction (FIXED-14 structural scoping): **the adversarial cross-tenant suite becomes the cross-org suite** - user of org A attempts every operation against org B's brand, knowledge, memories, artifacts, automations, Registo; clean 403/404 required. Plus in-org tests: builder A reads builder B's private memory → 404; edits B's private artifact → 403.
13. **Default design system.** Ships in-product: deliberate neutral palette, system font stack, no logo (header falls back to the org display name). Never the vendor's brand. `design-tokens.css` serves org tokens when research exists, the default otherwise. Migration: **one org per existing user, all on the default; no brand carry; the old company singleton is archived**. No user→org mapping file; the founder handles Brasil Salomão (brand research re-run, manual copies) post-launch.
14. **Teams deleted end to end** - endpoints, pages, stores, tests; glossary row added ("teams → deleted; departments, if ever demanded, return as additive groups inside an org"). Chapter 12's teams rows flip to delete.

## Part 6 - Per-build verification

17. Default ON for app builds: playwright-cli, medium depth, **incremental** - full acceptance pass on the artifact's first build, scoped tests of the change plus smoke on follow-ups. The verifying agent fixes forward within the slice retry budget; failure to fix completes the build with the honest visible note (carried mechanism). Tokens are `user_work`. UX: ask **once** on the user's first-ever build (answer stored as the setting; agent questions stay reserved for app ambiguity, never process); thereafter a banner while testing: "A testar a aplicação. Isto melhora a qualidade do resultado, mas torna a construção mais demorada e com maior custo. Pode desativar este comportamento nas definições da plataforma." Settings toggle per user. Live test-run streaming/click visualization: parked, post-launch.
